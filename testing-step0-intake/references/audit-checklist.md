# 产出真实性深度审核清单（配 SKILL §3.5）

> 总管"验真"模式用它，把"是否真做完"判到证据级。**核心立场：不信执行者自报，独立重算。** 实测教训：便宜/易偷懒模型常自报真测率 66% 实为 22%、整面空壳凑数、借图冒充、把分母从 1400+ 偷缩成 1073、面4 把 287 原子的分母换成 22 凑 100%。
> 三层：**① 机械层（确定性、无法橡皮图章）→ ② 语义层（读内容辨真假）→ ③ 重算真实口径**。只读：审产出文件 + 回链上游分母，不改不写库、不重做第①步探索。
> **形态裁剪**：纯 API/无前端形态下，UI/截图类强制产物与分母整组标 N/A、不当缺口（见 SKILL §3）。

## 一、各步"强制产物"清单（缺 = 未过闸）
先 Glob 该步产出目录，逐项核在不在、是不是空目录/空文件。**文件存在 ≠ 内容真**（内容真假交 §二/§三）。

### 审第①步（理解）
- `01-项目理解文档/` 下：总览架构 / 环境数据 / 认证权限 / 数据模型校验 / API导读 / **每业务模块各一份独立详解（不是挤进一个文件）** / 业务闭环与数据流图谱 / 产品文档对照与冲突 / 完整性自检与覆盖矩阵 / 测试就绪补全。
- `附-代码级勘察基线/`（A数据模型 / B-API与安全 / C前端页面 / D契约 等，带 file:line）、`附-扇出计划表.md`、各模块详解末尾《文件清点 已读X/X》。
- `02-证据/`（真截图，非空目录占位）、`03-冲突与待确认/`（独立成文）。
- **§5.5 产出后对抗审计**是否真做（有审计痕迹/反例）。

### 审第②步（用例）
- 各测试面目录的用例文件 + `00-用例编写规范` + `附-用例覆盖矩阵与追溯` + `附-扇出计划表` + 各块《格子清点》。
- 面4 文档符合性**独立成目录/文件**。
- **联动**：`附-联动点清单`（分母 N）+ `跨模块联动与变更影响/` 独立目录（联动用例）。
- **§8.5 产出后对抗审计**是否真做。

### 审第③步（执行）
- 各面**执行记录**（镜像第②步面/模块）。
- `执行进度跟踪.md`（各面/模块 真测/源码佐证/阻塞/截图覆盖，诚实口径）。
- `_执行真实性与截图覆盖-诚实审计.md`（§9 审计产出：真测率/反例）。
- `_第②步漏项反哺清单.md`、`_无法测试项说明(/汇总.md)`、`_失败用例/`（非空）、**独立的缺陷台账**（非只内嵌报告）、`01-测试环境与说明.md`、`测试报告.md`、`证据/`（真截图）。
> 实测反例：某次第③步顶层仅 `测试报告.md` + 空的 `_失败用例/`、`_缺陷台账/`，其余 8 个强制产物 Glob 全 0 命中 → 当场判未过闸。

## 二、机械层检查（确定性、可脚本化、无法橡皮图章）
工具：Glob 列文件、Grep 计数/找模式、`md5sum`/`Get-FileHash` 比图、`wc -l`/计数。中文路径用 Glob/Grep/Read（别用 PowerShell 回显中文会乱码）。

### a) 回链分母对账（"知道哪些没有"的核心——不重新理解项目，读上游已落盘的分母）
```
①理解  → 项目结构分母：模块数 / API N条 / 页面 M个 / 状态机 S个 / 业务闭环（"项目里有什么"）
   │  ↑回链
②用例  → 用例分母：每面每模块 N 条 TC + 面4 X 原子 + 覆盖矩阵（"该测多少"）
   │  ↑回链
③执行  → 实际执行了多少（"测了什么"）
```
- **数②用例分母**：`Grep '^### ' 第二步各面目录 -c` 求和（排除 README/规范/矩阵/扇出表等非用例文件）；面4 数对照表里的**原子 ID**（如 `#NN` distinct 计数）。
- **数③执行**：`Grep '^### ' 第三步执行记录 -c` 或数判定行；逐面对照②分母，**缺的 = 上层有下层无**。
- **抓"偷换/缩水分母"**：执行者自报"已执行 X / 真测率 Y%"——核 X 是不是 < ②真分母（如自报 1073 而②≈1400 → 缩水 300+）；核面4 自报"未测0/100%"时分母是不是被从 287 换成 22。
- **轻量分母可信度抽检**：①报"API=N"时，抽 2~3 个回**真代码 grep** 核数目对不对（仅抽检、不重做第①步）。

### b) 空证据 / 凑数 grep
- `Grep 'before=\s*$|after=\s*$|before=,\s*after=|实际结果:\s*$'`（证据为空的 FAIL/PASS）。
- 整批**同一句样板**当判定（如几十条都写"阻塞-需特殊条件(文件上传/浏览器/重启/破坏)"）→ 数它出现次数，占比高 = 空壳凑数。
- `Grep '源码佐证|未测|SKIP|阻塞|待补'` 计数 → 真测率分母里这些都**不算真执行**。

### c) 借图（截图造假）
- 对证据目录的截图算哈希：`md5sum *.png`（或 `Get-FileHash`）；**同字节哈希出现在多条不同声称下 = 一图冒充多条**（实测抓到 `DOC-02.png == DOC-03.png`）。
- 截图数：执行者自报 vs 实际 `*.png` 计数。

### d) 报告自相矛盾
- 报告自报"阻塞=A / 截图=B" vs 执行记录 `Grep '阻塞' -c` 实计、`*.png` 实数——对不上 = 报告统计不可信（实测自报阻塞41、实194）。

## 三、语义层检查（读内容辨真假；机械层判不了"内容真假"）
环境支持子代理时**派独立审核子代理**（干净上下文、立场怀疑、默认不通过）分块读执行记录/产出，逐项**举反例**；不支持则主循环对照本清单亲核、留判定依据。重点辨：
- **真测 vs 源码佐证**：判"通过/达标"的，证据是"真发请求带状态码+响应体 / DB 写后回读 / 浏览器真操作+专属截图"，还是"代码里有/逻辑存在"（= 源码佐证，按 step3 §8 不算真测）。
- **空壳**：联动"改A→看一眼没报错→回退"无下游断言锚点；持久化"回读"只在抄来的标题里、正文零 SELECT/查库证据；写失败(4xx)却判 PASS、证据全空却判 PASS。
- **借图/对题**：截图是不是本条专属、与被测点对应（不是拿"合法态图"冒充"脏数据兜底"、不是借他页）。
- **面4 三重**：每条文档声称是否都有 真实网页截图 + 位置说明 + API/函数真检测 + 逐条审计（缺一重不合格）。
- **覆盖深度**：UI 是逐控件真点/真填/真拖（可操作元素穷举），还是只"页面能打开/element-exists 探测"。

## 四、重算真实口径（不抄自报）
据机械+语义结果，给**真实分子/分母**：
- 真实真测率 = 真测条数 / **②真分母**（不是缩水后的执行集）。源码佐证/空证据/SKIP/阻塞 都不计入分子。
- 分面给：哪几面真达标（如 API/安全 真发请求扎实）、哪几面空壳（如联动真链 0/N、持久化写后回读 0/N、面4 真覆盖 X/总原子）。
- **如实肯定真的部分**（别一刀切否定）+ **戳破夸大**（自报 X% → 真实 Y%）。

## 六、边界与诚实
- 审核**只读**：审产出 + 回链上游已落盘分母；代码 grep 仅**轻量抽检**核分母，**不重做第①步探索**。
- 与各步内置 §5.5/§8.5/§9 **自审互补**：自审是第一道、易被懒模型自己跳过；本步是**跳不掉的外部兜底**（两终端经共享 `测试产出/` 目录协调，无需插件）。
- **天花板诚实标**：审核子代理也由主进程派出/采纳，机械层确定（无法橡皮图章），但语义层仍受执行方诚实约束；最终"要不要跑这套审核"的触发在执行者**之外**（人/CI），不能交给被审的模型。
