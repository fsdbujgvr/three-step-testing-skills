# 非标准形态的降级取证（次要·按需查阅）

> 核心方法是"代码 + 浏览器 + 前端渲染 + 文档对照"（见 SKILL.md）。绝大多数被测对象是带前端的 Web 应用，直接用核心方法即可。
> 只有当被测对象**不具备**核心方法所依赖的前提（有源码 / 有前端可渲染 / 有可登录账号）时，才查本文挂载对应降级手段——**目的不变：仍要产出"能写完整用例"的理解，只是换取证通道，并把够不到的面显式标为盲区**。

## 缺前提 → 降级出口
- **无源码（只给部署/只给 API）**：代码主线改走"黑盒契约"——抓 OpenAPI/Swagger、GraphQL introspection、抓包(HAR)、端点枚举/fuzz；结论标"契约推断"而非"源码确证"。
- **无产品文档**：显式记"无文档"，需求由代码+实测+界面反推；**不得因无文档判未完成**，DOC 证据级别下调。
- **无高权账号**：权限矩阵的"高权列"标"凭据缺失·残缺"，改做未认证/自助面 + 显式标盲区；不得假装测过 admin。

## 异形态 → 取证手段（前端渲染主线在此类失效，需替代）

### 纯 API / 后端服务（无前端，形态②）— 一节清单（替代前端八类/页面 M/M，成为第②步分母）
跳过"浏览器+前端渲染"两主线，以"代码/契约 + 请求矩阵实测"为主；前端八类/页面 M/M 闸门整组标 N/A，**改以下清单为完成度分母**：
- **契约勘察（逐端点逐字段）**：OpenAPI/proto 逐 path/方法/参数枚举——每个字段的 schema / 必填(required) / 枚举(enum) / 格式(format) / oneof / 默认值；响应信封结构；**错误码全集**（逐错误码 + 文案原文）。**端点 N/N、字段逐项**，这就是分母。
- **契约 ↔ 代码漂移取证**：契约声称 vs 代码实际路由/handler/校验逐条交叉核对，记下"契约有代码无 / 代码有契约无 / 字段口径不一致"的漂移项（带 file:line）。
- **认证形态**：API-key / mTLS（双向证书）/ JWT / 服务间(service-to-service) 鉴权——切身份 = 换 Authorization 头 / 换证书，逐角色取/签发凭据。
- **分页/过滤/排序口径**：page/size/cursor 风格、过滤参数全集、排序字段与默认序、越界/非法参数行为。
- **资源关系图**：实体间引用/外键/级联仍要连成图（同标准应用的闭环图谱），100% 入图。
- **取契约命令（示例）**：`curl <baseURL>/openapi.json`、`curl <baseURL>/swagger.json`、`grpcurl -plaintext <host:port> describe`、GraphQL introspection 查询；无契约文件时由抓包(HAR)/端点枚举反推并标"契约推断"。
- **交易/中台系统取证要点（形态③叠加，命中才用）**：
  - **并发/重放请求矩阵验幂等防超卖**：用少量并发请求（如 Promise.all 同时下单/扣减）+ 同一幂等键重放，验最终一致 / 不超卖 / 不重复扣款——**这是功能正确性，必真跑**，不是性能压测。
  - **对账只读回溯验最终一致**：挑一条真实交易链路只读拉出关联实体，核对金额/状态/余额守恒。
  - **支付/外部回调**：构造签名校验 / 重放 / 乱序到达场景，验回调处理的幂等与防伪。
  - **⚠ 写类验证仍遵 §3 安全红线**：上面"并发下单/扣减""回调重放"等涉及写库的真测，须在显式授权下用隔离 QA 数据 / 可逆往返进行；无授权则只做只读对账回溯、标"疑似·留授权阶段"，探索期默认不写部署库。

### 实时双工（WebSocket / WebRTC / SSE / gRPC-stream，形态④）
抓帧并回放、逆向帧语义/消息类型，枚举"消息/事件"而非"端点"；OpenAPI 对其失效，改用流量录制。
**实时面要摸清的清单（像 REST 端点清单一样成为第②步分母）**：
- **消息/事件 schema 与广播范围**：每类消息的字段 schema；广播到谁（房间/频道隔离边界——发到本房间还是全广播）。
- **协同冲突解决机制**：OT / CRDT / LWW(后写胜) 中的哪种——**直接决定第②步合并/冲突用例怎么写**。
- **连接与房间级鉴权**：建连鉴权、进房鉴权（越权进房？）、能否越权收到他人帧/他房间的数据。
- **重连补发/序号机制**：断线重连后是否补发漏帧、消息序号/去重机制、乱序处理。
- **presence**：在线状态来源、心跳间隔、失效（判离线）条件。
- **纯像素界面（Canvas / WebGL，无 DOM 文本）**："前端渲染"主线的 DOM 提取失效——改从前端内部状态(window/store/事件钩子)、交互事件流、或后端数据取证；核心交互（图元/光标/撤销栈/并发冲突）单列。
- **二进制私有协议 / 无界面后台（CLI / 批处理 / 消息驱动）**：从协议样本 + 日志 + 作业输入输出 + 消息 schema 反推；无 UI ≠ 无被测面。
- **Electron 桌面 / 移动端**：在浏览器主线外补——Electron 的 主进程/preload/IPC/本地存储/自动更新；移动端的 抓包/深链/WebView↔原生桥。

### 对话式 / LLM 应用（form-factor⑥：聊天机器人 / AI 助手 / Copilot / RAG，命中即叠加）
核心交互是"输入 →（流式）生成响应"，输出**非确定**——这是与传统系统最大的不同，决定断言方式必须换。三主线照常走（输入框/发送/消息列表/模型与参数选择器都是面1可操作元素；流式响应走上面"实时双工④"的 SSE/WS 取证；改配置→验生效是"建X验Y"闭环），**额外**挂载下面两块：
- **★非确定输出的断言范式（关键：不能断言"响应=精确文本"）**：LLM 同输入多次结果不同，传统"抄预期原文/断精确值"失效。改为**断属性 / 断结构 / 断不变量**：
  - 响应**非空且流式正常收尾**（收到结束事件/[DONE]、无中途断流）；
  - **格式/结构合规**（要 JSON 就真合法 JSON、要 markdown 就正确渲染、代码块语言标注正确）；
  - **不变量/护栏**：不泄漏 system prompt、拒答越权/有害请求、引用来源真实存在（RAG 防幻觉引用）；
  - **相关性而非等值**：响应"提及了关键实体/落在合理类别"，而非逐字匹配；
  - **非确定项多跑取分布**（呼应 step3 pitfalls 的 flaky）：同输入跑 N 次记"通过率/稳定性"，不把一次偶成当稳定、一次偶败当确定缺陷。
- **AI 应用特有测试维度（条件触发清单，命中才测）**：
  - **配置变更真生效**：切模型/调温度/换工具或 MCP/换 system prompt → 发消息后**查请求体真带了新配置**（前端选了 ≠ 真切了），且历史不丢；
  - **流式生命周期**：生成中"停止"真中断、停止后可继续/重发、重新生成产出不同、断网后重连；
  - **上下文记忆与窗口**：多轮引用前文、超长上下文截断策略（丢头/摘要）、新建会话隔离；
  - **安全/对齐**：提示注入（让它忽略指令/越权）、有害请求拒答、越狱尝试；
  - **附件/多模态（命中才测）**：上传文件/图片后响应真引用其内容、文件类型与大小限制；
  - **持久化**：对话历史/配置是会话级还是落库——按写后回读回读确认。
> 这一面是"叠加"而非替换：标准 Web 形态①的三主线照走，本节是额外挂载的"对话式专项面"，不抵消任何原有主线；纯后端 LLM API（无聊天 UI）则并入"纯 API②"，以请求-响应 + 上述非确定断言取证。

## 跨服务 / 多库
多服务·微服务·异步(MQ/事件)时，闭环用 correlation-id / 链路追踪 / 事件日志佐证；无法在单库一次只读回溯处，**显式标证据边界**。

## 收尾必做（无论何种形态）
在完成度闸门里增标一项：**本形态特有面（实时帧 / IPC桥 / Canvas交互 / 二进制协议 / 后台作业）已取证或显式标"盲区·原因"**——禁止把"够不到的面"默认成"不存在"。
