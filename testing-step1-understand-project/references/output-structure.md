## 命名与衔接要点
- 文件名用稳定序号/语义名，避免重命名破坏内部相对链接（移动整组时保持同级关系）。
- 「闭环图谱」「文档对照」「测试就绪补全」「完整性自检」是让"新模型立刻能写用例"的四块关键拼图，务必齐全。
- **★产出要对齐 skill 出现前那种"充分展开"的方式（不为省事压缩成少数几个文件）**：除上面的集成文档外，**每个业务模块各自单独成文**（不是把十几个模块挤进一个"各业务模块详解"），且 **02-证据 / 03-冲突与待确认（独立 4~5 份）/ 附-代码级勘察基线 A–D** 都要各自落盘。这不是凑文件数——而是单个文件装不下"详细到没见过的人能写最完整用例"所需的逐模块深度与逐条证据。**内容完整优先于文件数，但完整往往意味着自然展开成 20+ 份，而非合并成 10 份。**
- 各《05-模块-X.md》等子代理产出文件**末尾附该子代理《文件清点 已读X/X》**（含 glob/grep 命令原文与完整文件列表，SKILL §4 任务书⑤）；《扇出计划表》《文件清点》按上面目录树落盘，是**过程产物**（供 §5.5 审计追溯；**非 step0 进度完成信号、非第②③步用例/理解正文**，勿当成果扫读）。
- 冲突一律标明"以实测/代码为准"。**两处"冲突"载体分工要明确**：`01-项目理解文档/[文档]-产品文档对照与冲突.md` = 权威**逐条对照表**（文档声称 / 实测 / 判定，按维度穷举，是"哪些文档不可信"的单一权威源）；`03-冲突与待确认/冲突清单` = 供下游的**待办视图**：可引用 01 的权威对照表、但**本文件必须独立成文、不得为"去重"而省略或留空**（哪怕内容与 01-项目理解文档/[文档]-产品文档对照与冲突 有重叠也照样产出一份——这是用户明确要求的，宁多一份也要有）。待确认清单要把"此处只能猜"的雷区提前圈出，避免下游把猜测当确定。
- 第②③步的 README 先写"方针"占位（输入来自第①步哪些文件、覆盖哪些维度、性能指标清单），待上一步确认后再填充正文。

> 以上文件名中的 [闭环]/[文档]/[自检]/[就绪] 仅示意位置，落地时用具体序号（如 11/12/13/14）。

## 模块详解范例片段（领域中性 · 后台管理通用 — 示范"逐模块要挖多深"，非抄内容）
> 用"用户管理"这类任何后台都有的模块，示范单个《05-模块-XX.md》该有的深度。**学的是深度与结构，换项目按你自己的实体/字段/文案原文替换。**

### 模块：用户管理（后台）
- **入口**：路由 `/admin/users`(router/index.tsx:42)；列表 + 新建/编辑弹窗 + 删除二次确认。
- **数据模型**(users 表, models/user.py:10-28)：

  | 字段 | 类型 | 必填 | 唯一 | 约束/枚举 | 默认 | 前端校验(file:line) |
  |---|---|---|---|---|---|---|
  | username | str(50) | 是 | 是 | `^\w{3,50}$` | — | "用户名3-50位"(UserForm.tsx:31) |
  | role | enum | 是 | — | admin/operator/viewer | operator | 下拉、无自由输入 |
  | phone | str(20) | 否 | — | 手机号格式 | null | "手机号格式不正确"(UserForm.tsx:40) |

- **API**(users.py)：

  | 端点 | 方法 | 鉴权 | 副作用 | 错误码+文案原文 |
  |---|---|---|---|---|
  | /users | POST | admin | 建行 | 422"用户名已存在" / 422"用户名3-50位" |
  | /users/{id} | PATCH | admin | 改行(exclude_unset) | 404"用户不存在" |
  | /users/{id} | DELETE | admin | 软删 status=disabled | 404"用户不存在" |

- **状态机**：status active↔disabled；禁用后该用户登录返回 403"账号已禁用"(auth.py:55)。
- **越权风险**：operator 调任何 /users 写端点应 403；越权改他人 role 是高危点。
- **渲染对账**：role 走 statusMap(operator→"操作员")，未映射值兜底显示原文(需测脏数据)。
- **闭环**：建用户 → 该用户能否登录 → 能否到达其角色首页(建X验Y，跨认证模块)。
> 看这个深度：每字段带 类型/约束/校验文案原文+file:line、每端点带错误码原文、状态机带流转后果、还点出越权与渲染坑——**这才够第②步据此写出 边界/越权/状态/渲染 用例。十几个模块挤进一个文件、每个一两段，是达不到的。**
