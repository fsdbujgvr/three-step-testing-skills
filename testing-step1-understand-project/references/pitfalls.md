# 勘察避坑速查（工具/编码/认知陷阱·泛化）

> 这些是真实勘察中会绊倒模型、且与具体项目无关的坑。

## 工具稳定性
1. **单次工具结果过大 → 调用中断/解析失败**。根因是一次塞入过多数据（读十几页高清图、截十几张图、dump 超大 JSON）。对策：**小步**，每次读 2–4 页/少量图；工具调用与长篇写作分开；大列表只取计数+少量样本。
2. **全量 UI 快照极大**。需要交互定位时才取快照；**只为读文本/枚举时用 DOM 提取(evaluate)**，精确、紧凑、中文不乱码。
3. **看完即记录**：边勘察边把发现落盘，避免上下文压缩/会话重启后丢结论；长任务先落"骨架"再增量回填。

## 编码（中文/非 ASCII）
4. Windows/PowerShell 5.1 的 `Invoke-RestMethod` 取接口中文常**乱码**（按 Latin1 解码）。改用 `Invoke-WebRequest -UseBasicParsing` + `[Text.Encoding]::UTF8.GetString($r.RawContentStream.ToArray())` 再 `ConvertFrom-Json`。
5. PS5.1 `.RawContentBytes` 不存在（PS7 才有），用 `.RawContentStream.ToArray()`；不加 `-UseBasicParsing` 解析 2xx 会抛错。
6. 需要确切中文字符串时优先用浏览器/DOM 提取（渲染正确），而非命令行回显；写文件给其它工具读用 UTF-8。

## 浏览器自动化
7. 截图尺寸由**视口**决定（resize 到 1920×1080 即可）；物理窗口异常用 CDP 设窗口边界。
8. 验证"实时流/媒体是否真工作"：读 `<video>` 的 readyState/videoWidth/paused/currentSrc，比看截图更确证。
9. 切换登录身份：清 localStorage + 重新登录，或注入 token（需匹配用户档案结构）。
10. 用网络面板核对状态码；用 console 看运行期错误。

## 把"API 当证据"（高效且权威）
11. 边界/权限/越权用 **API 脚本化**比纯 UI 点击快且精确：跑"端点×身份"矩阵直接拿状态码与错误体；UI 截图用于视觉/交互证据。两者互补。
12. **越权写**用"创建→删除"可逆往返证明并**自清理**（先扫遗留再重试，避免污染被测库）；**写需显式授权**。
13. 管理员身份做**数据全景普查**（各列表计数+状态分布），快速建"线上实际有什么"基线、暴露脏数据。

## 认知陷阱（最容易"自以为做完"）
14. **特性开关改变"代码结论"的有效性**：代码里"启动迁移/种子"只在某开关下执行；线上没开则结论不成立——下结论前核对部署开关。
15. **文档/注释滞后于代码、代码又可能滞后于线上数据**——三者交叉验证，冲突以可复现实测为准。
16. **前端校验 ≠ 系统校验**：前端 rules 只是 UX，真正约束在后端；一端有一端无时必须两端都测、并说清后果。
17. **硬编码常量易被误读为数据**（型号、时间、版本号）——换一条数据看是否变化即可判定。
18. **未映射枚举的兜底渲染**会暴露"意外取值"（显示原始英文/空白）——凡按字典渲染处都构造字典外取值。
19. **UI 文案可能被运行期改写**：前端可能用 TreeWalker/MutationObserver 改 DOM 文本，**源码词≠显示词**——按显示文案断言、用路由/属性定位，别按源码文案 grep（搜不到时去入口文件找替换映射）。
20. **mock 端点 ≠ 真实数据**：有的"看板/统计"接口返回写死的假数据——换一次调用/换一条数据即可证伪。
21. **覆盖率"自答"会空转**：用"每个/全部"却没有分母 → 先穷举出分母（且用独立第二来源校验分母完备），再逐项标分子。
22. **把"看不到的面"默认成不存在**：实时帧/Canvas-WebGL/二进制协议/无界面后台没有 DOM/REST 可读 ≠ 没有被测价值——挂载对应手段或显式标盲区。

## 工具链 / 编排实战（让本步"一次性、无需提醒"跑通）
23. **取接口数据优先用 `node`（v18+ 全局 fetch，原生 UTF-8）或 `curl`，别依赖 PowerShell/python**：Windows PS 5.1 `Invoke-RestMethod` 中文按 Latin1 乱码；自定义函数 + 字节体 POST 可能出现"completed with no output"诡异静默；git-bash 里 `python`/`python3` 常被 Windows 应用别名劫持不可用。JSON 解析也用 node。`.mjs` 用 `import` 不能 `require`。
24. **子代理(subagent)扇出结果别灌进主上下文**：让每个子代理返回纯 markdown（约定从首个 `# ` 标题截取、去寒暄），主循环用脚本把扇出返回结果按 label 拆分落盘（**Claude Code Workflow 返回 `{summary,agentCount,logs,result:[...]}`、取 `.result` 拆分；OpenCode/Codex 等终端按其子代理返回结构类推——每个子代理返回结果、主循环汇总拆分落盘**）。代码线穷举（后端/前端两线起步，可再按 模型/API/前端页面/基础设施 细拆——与正文 N+3 下限+上不封顶口径一致）与各业务模块详解都适合这样扇出；**浏览器线不可并行**（单一有状态实例会冲突），留主循环串行；总览/闭环/文档对照/自检/就绪等集成文档由持全局上下文的主循环亲写以保一致。
25. **在浏览器页面上下文里用已登录态 `fetch` 做 API 取证/真实链路回溯**：读 localStorage 的 token 调 API（`fetch(u,{headers:{Authorization:'Bearer '+localStorage.getItem('accessToken')}})`）——天然带鉴权、中文不乱码，最省事地核对 owner/status 贯穿。
26. **Playwright 浏览器实例/配置目录易被残留进程锁**（"Browser is already in use … use --isolated"）：按命令行含 `mcp-chrome-<id>` 用 `Get-CimInstance Win32_Process` **精确 kill 残留进程**（勿误杀用户正常浏览器），再重连。
27. **截图落点可能 ≠ 预期目录**：`take_screenshot` 的相对文件名可能落到**当前工作目录根**而非 MCP 输出目录；用 `find -newermt` 定位真实路径后再归集进证据目录。
28. **恢复中断的长任务先核验盘上现状**：进度记忆是"当时真"的快照，可能已与磁盘分叉（如产出目录被清空但记忆仍说"已完成"）——**以盘上现状为准**，不盲信记忆。
29. **"代码暴露" ≠ "入口可达"**：后端可能开了 /docs、/openapi.json，但反代/SPA 在公网入口接管/遮蔽——在**真实对外入口**验证可达性，而非只看源码下结论。
