# 测试面 × 测试维度 矩阵（展开 · 条件触发 · 贯穿维度）

> 配合 SKILL.md 第 1 节。这是覆盖的"分母骨架"：用例 = 面 × 维度 应用到某模块/特性，内容由第①步发现填充。**面/维度通用，内容项目特有**。下文 🟦=通用恒考虑 / 条=条件触发（命中项目特征才加）。examples 仅示意，不要照抄成结论。

## 一、7 个测试面（轴一）逐面展开

### 面1 · UI 功能与闭环（浏览器/前端渲染 · 所见与可用）🟦
- 功能正向 + 可用性：每页/每控件/空态/加载态/错误态/分页/弹窗/二次确认文案。
- **交互方式穷举（🟦贯穿 · 命中即必测）**：对每个控件**先判定它支持哪些交互方式**（单击/双击/右键/悬停/拖拽/滑动/拖放/长按/快捷键/滚轮）——
  - **默认硬性（恒必测）**：每元素 × 其默认支持的交互（单击/输入/选择/键盘可达）各建用例。
  - **富交互（命中即必测、命中才进分母）**：拖拽绘制、悬停(缩放预览/tooltip)、滑块拖到边界(min/max/越界/反向)、右键菜单每项、文件拖放、弹窗内部控件 等——**仅当该控件确实支持该交互时即必测**，命中时逐方式各建一条可独立断言的用例、一个都不省（Canvas/富编辑器只是其加重场景、非唯一入口）；有滑块/悬停/右键但无 Canvas 的页面（可视化/媒体/上传/后台管理类）最易两头落空漏测。
  - **整体豁免**：本页/本系统控件仅点击/输入/选择、无富交互时，一句"无富交互"整体豁免该组，不必逐控件标不适用。
  - **形态边界**：本组**仅适用于有 UI 的形态**；纯 API/无前端形态整组标 N/A（见 SKILL §0.1）。
- **功能闭环（跨模块）**：此处操作 → 去**别处**(列表/统计/其它角色/其它页)验证记录生效/同步/级联——这是端到端用例的核心，不可只孤立测单模块。
- 列表交互：搜索(每个可搜字段)/筛选组合/排序/重置/空结果。
- 状态/枚举/文案**展示**与**未映射兜底**（见贯穿维度·鲁棒性）。
- 条：富交互/画布（见面外条件专项）、实时流/音视频 UI、导出/打印的 UI 触发。

### 面2 · 后端接口与数据（实现真相 · 返回与落库）🟦
- 每端点：方法/鉴权/入参；正向返回(信封正确) + **DB 写入/副作用/级联**；每条错误分支(状态码+文案原文)。
- 输入边界/超出数据：空/超长/特殊字符/注入样本/emoji/中文/最值±1/格式正则/唯一重复/默认。
- **业务规则与计算正确性**（见贯穿维度）：聚合/统计/派生值/序号/时长 的公式结果。
- **并发与数据完整性**（见贯穿维度）。
- 分页口径(参数名/默认/上限/真实总数是否截断)。
- 条：文件/对象存储链路、外部服务端点(AI/支付/地图)。

### 面3 · 前后端通讯（请求方式与一致性）🟦
- 请求异常：超时/重试/超限(网关与后端各一道)/并发/畸形请求(非JSON/缺头/超长头)/分页参数。
- **绕过前端校验直打 API**：前端 rules 只是 UX，真约束在后端；同字段两端规则差异、一端无校验的后果。
- 弱网/离线/恢复（见贯穿维度·恢复）。
- **前端态 vs 真落库**：UI 动作 → 确认实际请求(方法/路径/body) → 响应信封处理 → 回读确认真入库；辨"建议≠保存""勾选是否真发请求"。
- **形态分流**：上面"绕过前端校验直打 API""前端态 vs 真落库"等**前后端对比项**仅在有前端时适用；**纯 API/无前端形态下本面收敛为"契约一致性 / 超时重试 / 畸形请求"**（契约↔实现一致、超时重试幂等、缺头/非JSON/超长头），不做前后端校验对比（见 SKILL §0.1）。

### 面4 · 文档符合性（★独立面 · 声称真相）— oracle = 外部需求文档
- 判定基准与其它六面**本质不同**（对账"声称"而非测行为），故**独立成面/文件夹**。
- 每条声称/规格 → 正向核验；每个操作步骤 → 走查；声称却未实现 → 负向核验。
- 有官方测试记录单/报告 → 逐条 1:1 映射成覆盖表（详见 pitfalls.md「文档符合性映射」）。
- 条：仅"有产品/需求文档"时启用（无文档则显式记"无"，不判未完成）。
- **形态分流（眼见为实的取证手段随形态变）**：标准 Web 应用——"应有 UI 的声称"强制浏览器截图，**不可只源码佐证替代**；**纯 API/无前端形态**——对纯后端规格声称（幂等/事务/金额公式/状态机/契约字段）改以**"真实请求-响应 + schema/契约校验 + 代码 file:line"为"眼见为实"**，"必须浏览器截图、不可源码佐证替代"整组标 N/A（见 SKILL §0.1）。

### 面5 · 权限/越权/认证/会话 🟦
- 权限矩阵：每个受保护资源 × {无token/各低权角色/高权} 三态状态码。
- 前端路由守卫 vs 后端接口鉴权 是否对等。
- 越权读(低权读他人)/越权写(匿名或低权 增删改受保护数据，**写需授权·可逆往返**)/数据隔离(列表按 owner 过滤)。
- 角色字段可否越权篡改(无枚举约束的 role)。
- **会话/Token 生命周期**：过期/刷新/登出清存储/多端同步/role 改库即时生效/坏token/记住。
- **RBAC 配置正确性（把权限当业务功能测；有可配权限/角色管理即触发）**：对每个权限点做**正反对照**——授予→该角色**可见可用且后端放行**；收回→**入口消失且后端 403**（不能只前端隐藏、后端仍放行）。三粒度各至少一条：**菜单级**（导航/页面入口）、**按钮级**（操作按钮可见可点）、**数据范围/行级**（只看到自己范围的数据）。并验**权限变更生效时机**：改权限后是否需重新登录/刷新 token 才生效（即时生效 vs 旧会话仍持旧权限）。

### 面6 · 安全与配置 🟦
- 无鉴权高危端点清单(匿名读/写)。
- 密钥/凭证明文暴露(配置/前端)。
- CORS/CSP/安全头(X-Frame-Options/nosniff/HSTS/Referrer/Permissions-Policy)。
- 错误信息是否泄露内部(堆栈/SQL/路径)。
- 静态目录/代理类接口的遍历与 SSRF/滥用面。
- 条：媒体/第三方隐私(上传可见性/数据出境)。

### 面7 · 数据持久化与端到端深度 🟦
- **写后回读**（铁律，见 SKILL §4 + pitfalls.md）。
- 多步操作**原子性/回滚**：前端串行多写中途失败是否留孤儿数据。
- 级联删除回读消失；外键置 NULL 回读。
- 复杂闭环**逐步回读**(每步验该步 DB 效果)：典型链路从头到尾，枢纽实体串起各表状态一致。
- **含回环的链路要列全、每条走通一遍**：能回到先前状态的环路(打回→整改→再提交、提交→驳回→再改 这类，常跨角色)**逐条枚举进《回环清单》**，每条环路至少一条端到端用例走通一遍即可——**不重复循环 N 次**(多遍执行耗资源无额外价值)。这是"完全统计项目所有回环"的落点(见 SKILL §3「状态机回环完全统计」+ §8 闸门)。

## 二、轴二 · 横切维度（逐面施加，别只放一处）🟦
| 维度 | 说明 |
|---|---|
| 功能正向 | 主路径按预期 |
| 边界/超出 | 空/超长/特殊字符/最值±1/格式/唯一/默认/超大payload |
| 状态机 | 每状态允许操作；合法+**非法**流转(错误来源态触发动作)；**回环/环路完全统计**：列出所有"能回到先前状态"的环路(如 待审→打回→整改→待审)，每条**走通一遍**(非重复 N 次)，环路清单来自第①步状态机图谱 |
| **计算/业务规则正确性** | 喂已知输入→断言聚合/统计/序号/时长/比率 **公式结果精确**，非只验"有显示"；注意脏数据导致的异常值(如负数) |
| **并发与数据完整性（=功能必测，必真跑）** | 少量并发请求验**最终一致 / 防超卖 / 幂等去重 / 防重复提交 / 归属抢占 / 丢失更新**——这是**功能正确性缺陷类**，与吞吐压测**不是一回事**，**必须真跑**，不得归入留③/阻塞/"源码佐证即可"。**幂等/防超卖触发条件：凡有唯一约束或库存/计数类字段即触发**（非电商专属） |
| **展示鲁棒性/脏数据** | 未映射枚举兜底/**无兜底白屏**;字典外取值;null/负值/乱码渲染;错误文案与提示类型一致;状态徽章跨模块一致 |
| **错误恢复/原子性** | 多步失败孤儿数据;弱网离线;服务重启;刷新/后退/深链/未保存拦截 |
| 权限越权 | 见面5 |
| **写后回读持久化** | 见面7/SKILL §4 |
| 编码/本地化 | 中文/emoji/特殊字符/全半角;导出 BOM;时区/日期格式 |
| 兼容性 | 浏览器版本矩阵/目标分辨率/缩放/响应式 |
| **性能：吞吐/时延（留③）** | 吞吐/响应/时延/耗时/压测——本步定义指标与判定法，执行属③。**注意：这一行只是吞吐压测，与上面"并发与数据完整性"的最终一致/防超卖正确性验证彻底分开，别把后者误归本行而漏跑** |

## 三、条件触发面/专项（按第①步"项目形态特征"勾选）
| 若项目有… | 触发专项 | 要点 |
|---|---|---|
| 实时流/音视频 | 媒体专项 | 连接/断开/重连/降级;无能力浏览器;设备权限;端到端 |
| AI/外部服务 | 第三方专项 | 成功/失败/超时/弱网/配额/隐私;返回是否引用已删资源 |
| Canvas/富编辑器 | 画布深验 | 每工具实绘/撤销删除清空/快捷键/最小尺寸/导出参数/保存是否入库 |
| 导出/打印 | 导出保真 | 导出物真含正确内容/图/硬编码值/多页/中文/文件名/BOM |
| 文件/对象存储+历史迁移 | 存储多链路 | 新旧链路渲染分支;URL 解析;代理;CORS;迁移一致性 |
| 产品/需求文档 | 面4 文档符合性 | 逐条对照 + 官方记录单映射 |
| 多用户 | 并发加重 | 见并发维度 |
| 统计/派生值 | 计算正确性加重 | 见计算正确性维度 |
| **有唯一约束/库存/计数/金额字段** | **并发/幂等正确性专项（必真跑，非压测，不归留③）** | **幂等键**→重复 POST 两次断言**只建 1 行 + 第二次返回同一资源**（非新建）；**库存/计数并发扣减**→并发 K 个请求断言**不超卖、失败者得 409 / 库存不变**、成功数 == 可用量；**重试副作用**→超时后客户端重试断言**无重复扣款/无重复创建**（服务端去重）。每条均"写后回读"守恒断言（见 SKILL §4 重复写后回读）。 |
| **有 WebSocket / 实时协同 / presence / 通知流** | **实时协同专项（与音视频媒体专项明确区分：那是 RTC 流，这是消息语义）** | **连接生命周期**：握手鉴权/心跳/超时/重连/**重连后补发漏帧**；**并发协同合并**：OT/CRDT/LWW 结果、光标/选区冲突；**消息顺序与幂等**：乱序/重复/丢帧处理；**presence 准确性**：异常断开是否仍显示在线/多标签页计数；**通知流**：投递保证/已读态/离线补发。双 oracle：DB 回读 + 第二在线客户端收广播（见 SKILL §4 实时增量写）。 |
| **有批量导入 / 导出** | **导入导出对账专项** | **导入** = 合法+非法行混合文件 → **事务策略**（全回滚 vs 跳过坏行）符合声称 + **错误回执定位到行/列** + 成功行**写后回读** + 中文/编码正确；**导出** = **逐字段对账真值**（非只验"有文件") + 筛选条件带入导出 + BOM/中文不乱码 + 大数据量是否截断/分页/分片。 |
| **对话式 / LLM 应用（聊天/AI助手/Copilot/RAG，form-factor⑥）** | **LLM 应用专项（输出非确定，断言方式必须换）** | **★非确定断言**（不抄精确响应文本）：响应非空+流式正常收尾([DONE]/无断流) / 格式结构合规(要JSON就合法·markdown正确渲染) / 不变量护栏(不泄漏 system prompt·拒答有害越权·RAG 引用来源真实存在) / 相关性而非等值(提及关键实体即可) / 同输入多跑取分布(记通过率,不把偶成当稳定)；**AI 特有维度**：配置变更真生效(切模型/温度/工具→**查请求体真带新配置**,前端选了≠真切了,且历史不丢) / 流式生命周期(停止真中断·重新生成·断网重连) / 上下文记忆与窗口截断 / 提示注入·越权拒答 / 附件多模态真引用 / 对话历史与配置持久化(写后回读)。详见 step1 `references/non-standard-forms.md` 对话式节。 |

> 此表**只是举例、不穷举**：按第①步发现的**领域特性**派生本项目的高负载专项，与上表并列。例如电商/交易类应补：支付回调幂等与重复扣款、价格/优惠叠加/退款金额公式精确、库存并发扣减防超卖、订单状态机非法流转；社交类补：关注/拉黑/隐私可见性/信息流排序；**实时协作类**（在线文档/白板/聊天/看板）应补：连接生命周期与重连补发、协同合并冲突(OT/CRDT/LWW)、消息顺序去重、presence 准确性、通知投递保证；这些可由通用维度(计算正确性/并发/状态机/越权)涵盖，但要从第①步主动识别为"重负载点"单列，别等同一行通用维度带过。专项目录名按本项目语义起，勿照搬别项目(如"画布深验"仅在有画布时才用)。

## 四、贯穿维度提醒（最易"只放一处就忘")
计算正确性 / 并发 / 鲁棒性·脏数据 / 错误恢复 / 编码本地化 / 越权 —— 这些会**跨多个面复现**。主层挂靠后，在其它相关面也要施加并交叉引用（如"持久化"面的回读用例与"并发"维度结合 → 并发下回读一致性）。

## 五、关键认知
- **"通用 vs 特有"的真分界线是"维度 vs 内容"**，不是"哪几号分类"。绝大多数"看似新增的分类"其实是通用维度/条件面对核心面的深化——"补得晚"≠"项目特有"。
- 换个项目时：7 面骨架恒在(面4 视有无文档)，横切维度逐面过，条件面按其特征触发，内容全部由该项目第①步发现填充。这样不会因"某维度是上个项目才加的"而漏掉它。
